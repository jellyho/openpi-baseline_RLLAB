§1 · ACSAC in brief

ACSAC learns a critic that scores every prefix of an action chunk and selects the execution horizon per state. Four equations carry the whole method.

An action chunk of length \(H\) is \(a_{t:t+H}=(a_t,\dots,a_{t+H-1})\); its length-\(h\) prefix is \(a_{t:t+h}\). The prefix value is the return of executing \(h\) actions open-loop then following the policy — for every \(h\) it is "total discounted return from \(s_t\)", which is what makes the values comparable across horizons (ACSAC Thm G.9):

$$ Q^\pi(s_t,\,a_{t:t+h}) \;=\; \underbrace{\sum_{\tau=0}^{h-1}\gamma^\tau r_{t+\tau}}_{r^{(h)}_t}\;+\;\gamma^h\,V^\pi(s_{t+h}). $$

Per-horizon target (ACSAC Eq. 5): one target per prefix length, bootstrapping at the matching next state \(s_{t+h}\):

$$ G_h \;=\; \sum_{\tau=0}^{h-1}\gamma^\tau r_{t+\tau} \;+\; \gamma^h\,\bar Q\!\big(s_{t+h},\,\pi^\star(s_{t+h})\big). $$

Multi-step TD loss (ACSAC Eq. 6) — the \(H\) squared errors are averaged at the gradient level (the \(\tfrac1H\)), which reduces variance while preserving sparse-reward signal:

$$ \mathcal{L}_{\text{reg}}(\phi) \;=\; \mathbb{E}\!\left[\frac{1}{H}\sum_{h=1}^{H}\big(Q_\phi(s_t,a_{t:t+h})-G_h\big)^2\right]. $$

Expected-prefix-max bootstrap (ACSAC Eq. 19) — a generalization of EMaQ: sample \(N\) chunks from the behavior policy at \(s'\), take the max over all \(N\times H\) prefixes, with a \(\min\) over the \(K\)-critic ensemble:

$$ V^\star(s') \;=\; \max_{n\in[N],\;k\in[H]}\;\min_{j\in[K]}\;Q_{\phi,j}\!\big(s',\,\tilde a^{(n)}_{s':s'+k}\big). $$

Adaptive extraction (ACSAC Eq. 8) — the same rule for the bootstrap action and for deployment; the executed horizon \(h^\star\) is state-dependent:

$$ (n^\star,h^\star)=\arg\max_{n\in[N],\,h\in[H]} Q_\phi\!\big(s_t,a^{(n)}_{t:t+h}\big),\qquad \pi^\star(s_t)=a^{(n^\star)}_{t:t+h^\star}. $$
ACSAC는 ① 모든 prefix의 가치 \(Q(s,a_{1:h})\)를 학습하고(같은 단위라 비교 가능), ② horizon별 target \(G_h\)로 회귀하며(\(\tfrac1H\) 평균), ③ bootstrap은 \(N\times H\) prefix에 대한 max(앙상블은 min), ④ 배포 시 \((n,h)\) joint arg-max로 \(h^\star\)만 실행합니다.

§2 · Concept primer

2.1 · Why distributional?

Instead of regressing a scalar \(Q\), a distributional critic predicts a categorical distribution over a fixed value support and is trained by classification (cross-entropy). "Stop Regressing" (Farebrother et al., 2024) shows this is more stable and scales better than MSE for value learning — the bounded, normalized targets behave better than raw regression, especially with deep nets and long horizons. Our reference repo DEAS-FQL uses exactly this (HL-Gauss).

2.2 · HL-Gauss (histogram loss with a Gaussian)

Fix a support \([v_{\min},v_{\max}]\) with \(M\) bins, edges \(z_0<\dots

$$ p_\phi=\operatorname{softmax}(\ell_\phi),\qquad Q_\phi=\sum_{i=1}^{M} p_{\phi,i}\,c_i \quad(\texttt{transform\_from\_probs}). $$

A scalar target \(y\) is turned into a soft two-hot label by integrating a Gaussian \(\mathcal{N}(y,\sigma^2)\) over each bin (\(\Phi\) = standard normal CDF):

$$ p^{\text{tgt}}_i(y) \;=\; \frac{\Phi\!\big(\tfrac{z_{i+1}-y}{\sqrt2\,\sigma}\big)-\Phi\!\big(\tfrac{z_{i}-y}{\sqrt2\,\sigma}\big)}{\Phi\!\big(\tfrac{z_{M}-y}{\sqrt2\,\sigma}\big)-\Phi\!\big(\tfrac{z_{0}-y}{\sqrt2\,\sigma}\big)}\quad(\texttt{transform\_to\_probs}). $$

The denominator renormalizes over the truncated support. This is implemented numerically-stably in utils/distributional.py (ported from DEAS's utils/hlg.py).

HL-Gauss trains a categorical representation of a scalar regression target — the target is the scalar \(G_h\) turned into a soft histogram, not a full distributional Bellman backup (no projecting the next-state distribution through \(\gamma\) and \(r\)). This is exactly what DEAS does and what we asked for; it keeps the bootstrap a single scalar \(V^\star\) and reuses ACSAC's machinery unchanged.
분포형 critic은 스칼라 \(Q\) 대신 고정 support 위의 categorical 분포를 출력하고 cross-entropy(분류)로 학습합니다. HL-Gauss는 스칼라 target \(y\)를 가우시안으로 부드럽게 만든 히스토그램(\(p^{\text{tgt}}\))으로 바꾸고, 값은 \(\sum_i p_i c_i\)로 읽습니다. 이는 DEAS와 동일한 방식이며, target은 여전히 스칼라 \(G_h\)라 ACSAC 파이프라인을 그대로 재사용합니다.

The Transformer critic, explained ★ (no prior transformer experience needed)

Our critic is a small causal Transformer (utils/transformer.py). This section builds it up from zero — what a transformer is, why we use one here, and a line-by-line read of the code.

Why a transformer at all?

We want one network that, given a state and an action chunk \(a_{t:t+H}\), outputs the value of every prefix \(Q(s,a_{t:t+h})\) for \(h=1,\dots,H\) at once, with one hard requirement: the value for prefix \(h\) must depend only on the first \(h\) actions — not on the actions that come after. A plain MLP fed the whole chunk violates this (its output for "the first 3 actions" would secretly depend on actions 4..H). A transformer with a causal mask enforces it by construction. That property — prefix consistency — is what makes comparing values across horizons (and the adaptive \(h^\star\) selection) valid.

우리는 한 번의 forward로 모든 prefix의 가치 \(Q(s,a_{1:h})\)를 출력하되, prefix \(h\)의 값이 앞쪽 \(h\)개 행동에만 의존해야 합니다(뒤 행동에 의존하면 안 됨). 일반 MLP는 이걸 보장하지 못하지만, causal mask를 쓴 Transformer는 구조적으로 보장합니다. 이 성질(prefix consistency)이 horizon 간 값 비교와 적응적 \(h^\star\) 선택을 가능하게 합니다.

A transformer in four ideas

IdeaWhat it means here
1. TokensWe turn each input piece into a vector ("token"). Here: the state \(s\) becomes one token, and each action \(a_{t+i}\) becomes one token. So the sequence is \([\,s\,|\,a_t\,|\,a_{t+1}\,|\dots|\,a_{t+H-1}\,]\), length \(L=H+1\). A learned positional embedding is added so the model knows each token's slot.
2. AttentionThe core operation. Each token produces a query, and every token a key and a value. A token "attends" to others by comparing its query to their keys (dot-product → softmax weights), then takes the weighted average of their values. Intuition: a soft, learnable lookup — each token pulls in information from the tokens most relevant to it.
3. Causal maskWe forbid a token from attending to later tokens — token \(i\) may only look at tokens \(0,\dots,i\). So the action-\(h\) token can see the state and actions \(1..h\), but never \(h{+}1..H\).
4. BlocksWe stack a few identical blocks, each = (attention) + (a small per-token MLP), wrapped with LayerNorm (normalizes a vector) and residual add (output = input + sublayer, which stabilizes training). Two blocks here.

The picture

tokens:     [   s   |  a_t  | a_{t+1} | a_{t+2} | ... | a_{t+H-1} ]      length L = H+1
position:       0       1       2        3              H
                                  causal attention (who can look at whom)
        s        : ■
        a_t      : ■  ■
        a_{t+1}  : ■  ■  ■

Because the \(h\)-th action token only ever mixed in the state and the first \(h\) actions, the value we read there is a clean function of the length-\(h\) prefix — exactly the prefix consistency we needed.

Sizes & the ensemble

Two attention layers, 8 heads, head-dim 16 → token width \(128\); the per-token MLP is width 512. The whole critic is tiny (that is why ACSAC has the smallest parameter count of all baselines — one shallow shared backbone serves every prefix length). PrefixValue (transformer.py:144) wraps this with ensemblize(K) to make \(K{=}2\) independent critics in one vmapped call, so every output above gains a leading ensemble axis \((K,\dots)\); we take the \(\min\) over \(K\) for the target/extraction.

Read the value off the action token, not the state token. The code slices x[..., 1:, :] (:114) — dropping position 0 (the state token) — so the head at position \(p\in\{1..H\}\) gives \(Q(s,a_{t:t+p})\). An off-by-one here would silently shift every horizon.
The distributional change (§3) touches only the last line: Dense(num_atoms) instead of Dense(1). Everything in this section — tokens, attention, causal mask, blocks — is shared by both the regression and distributional critics, and by the OGBench and frozen-VLA settings alike.
구현 요약: ① state·action을 각각 토큰(벡터)으로 만들고 위치 임베딩을 더함 → ② causal mask를 쓴 attention + MLP 블록을 2번 통과(Pre-LN, residual) → ③ 각 action 토큰에서 출력 head를 읽어 prefix별 \(Q\)를 얻습니다. 분포형 critic은 마지막 head만 Dense(1)→Dense(num_atoms)로 바꾼 것뿐이고, 나머지(토큰·어텐션· 마스크·블록)는 전부 동일합니다. 앙상블은 PrefixValue가 \(K{=}2\)로 감쌉니다.

§3 · The distributional critic ★

The one modification, precisely: swap the per-prefix head and the loss; everything else (transformer backbone, expected-prefix-max, extraction) is untouched.

What changes in the network

The causal Transformer still emits one output per action token, but that output becomes \(M\) logits instead of a scalar. So the critic returns, per prefix \(h\),

$$ \ell_\phi(s_t,a_{t:t+h})\in\mathbb{R}^{M}\quad\text{(shape } (K,B,H,M)\text{)}\qquad\text{vs. regression } Q_\phi\in\mathbb{R}\ \ (K,B,H). $$

Code: one line — Dense(num_atoms) at each action token, squeezed to scalar only when num_atoms==1. utils/transformer.py :118–121.

What changes in the loss

The target is the same scalar \(G_h\) (Eq. 5), but the prediction is now matched by cross-entropy against its HL-Gauss soft label, still averaged over horizons:

$$ \mathcal{L}_{\text{dist}}(\phi)=\mathbb{E}\!\left[\frac{1}{H}\sum_{h=1}^{H}\operatorname{CE}\!\Big(p^{\text{tgt}}(G_h),\,\ell_\phi(s_t,a_{t:t+h})\Big)\right],\quad \operatorname{CE}(p,\ell)=-\sum_{i=1}^{M}p_i\log\operatorname{softmax}(\ell)_i. $$

Code: agents/aqc.py critic_loss():128, distributional branch :153–156 (target_probs = _scalar_to_probs(G_h), ce = categorical_cross_entropy(pred, target_probs)).

What stays the same (and why it just works)

The bootstrap and extraction operate on expected values, so they are identical to regression once we read \(Q_\phi=\sum_i p_{\phi,i}c_i\). The expected-prefix-max becomes

$$ V^\star(s')=\max_{n,k}\;\min_{j}\;\sum_{i=1}^{M}\operatorname{softmax}\!\big(\ell_{\phi,j}(s',\tilde a^{(n)}_{:k})\big)_i\,c_i. $$

Code: a single helper agents/aqc.py _prefix_values():60 returns the expected scalar for either critic type; _expected_prefix_max, sample_actions, and sample_actions_with_info all call it, so none of them needed branching.

§5 · Training setup (offline, frozen VLA)

This is where our setting differs from the ACSAC paper. The paper jointly trains the flow-BC policy and the critic online. We have a frozen VLA (Pi0.5) and train critic-only, offline, from precomputed data.

The mapping to ACSAC

flow-BC policy \(\pi_\theta\)the frozen VLA (Pi0.5 flow-matching); not trained, only sampled from
state \(s_t\)\(s_t=\text{latent}\in\mathbb{R}^{2048}\) — the RL-token embedding, which now fuses the 3 cameras and proprioception (the VAE ingests proprio), so the critic input is the latent only (no separate proprio token)

So the transition \((s,a,r,s',a')\) maps as: \(s=\text{latent}_t\); \(a=a_{t:t+H}\) (the executed behavior chunk); \(r=\sum_{\tau} r_{t+\tau}\) (per-step rewards summed; ACSAC uses the discounted \(r^{(h)}_t\) per prefix); \(s'=s_{t+h}\) (the per-prefix next-state latents); and \(a'\) = the precomputed VLA candidate chunks at \(s'\).

Reward & value support \([v_{\min},v_{\max}]\)

The reward is sparse-style: a success trajectory is \(r=-1,-1,\dots,-1,0\) (the \(0\) at the success step); a failure trajectory is \(r=-1,\dots,-1,-C_{\text{fail}}\) with a large penalty \(C_{\text{fail}}>0\) at the failure step. Rewards are then normalized so the value lands in \([-1,0]\). Hence for the distributional critic we set

$$ v_{\max}=0\quad(\text{best: immediate success}),\qquad v_{\min}=-1\quad(\text{worst: normalized failure return}). $$

so the HL-Gauss support is \([-1,0]\) (a small margin, e.g. \([-1.0,\,0.0]\) with \(M{=}101\) atoms, is fine).

No discount (\(\gamma=1\)), following RECAP. RECAP (\(\pi^\star_{0.6}\), Physical Intelligence) defines the return as the undiscounted sum \(R(\tau)=\sum_{t=0}^{T} r_t\) ("we do not use a discount factor") and a distributional value over \(B{=}201\) bins trained by cross-entropy — the same family as our HL-Gauss. So for this setting we set --discount=1.0, v_min=-1, v_max=0, and (to match RECAP) num_atoms=201. The return is bounded because episodes terminate (the done flag zeroes the bootstrap), and is normalized into \([-1,0]\).
\(\gamma=1\) trades away the contraction guarantee — prefer \(\gamma=0.9999\). ACSAC's contraction proof (Thm G.13) needs \(\gamma<1\). With \(\gamma=1\) and a TD bootstrap this is lost. The clean fix is --discount=0.9999: the effective horizon \(1/(1-\gamma){=}10{,}000\) steps \(\gg\) the ~2000-step episodes, so it is effectively undiscounted over an episode (matching RECAP) yet the Bellman backup is a proper \(\gamma\)-contraction. The \([-1,0]\) support is unaffected (discounting only makes returns less negative). Use \(\gamma=1.0\) for an exact RECAP match, or \(0.99\) if you want stronger stability. Either way, HL-Gauss puts zero mass outside \([v_{\min},v_{\max}]\), so verify with the return-histogram check (§7) that <1% is clipped.
Why TD, not Monte-Carlo. RECAP trains its value by MC (target = empirical return-to-end). That cannot drive adaptive chunking: the MC return-to-end from \(t\) is the same regardless of prefix length \(h\) (the trajectory already continued under the behavior policy), so \(Q(s_t,a_{t:t+h})\) would be identical for all \(h\) — no \(h^\star\) signal. ACSAC's per-prefix \(G_h=r^{(h)}_t+V^\star(s_{t+h})\) is exactly RECAP's advantage n-step term \(\sum_{t'=t}^{t+N-1}r_{t'}+V(o_{t+N})\), made per-prefix and action-conditioned. The advantage \(A=Q(s,a_{t:t+h})-V(s)\) is the bridge to RECAP-style advantage-conditioning for downstream VLA policy extraction.

Critic-only objective

Since the VLA is frozen, there is no actor / flow-BC loss — we minimize only the (distributional) per-prefix TD loss of §3. The bootstrap reads the precomputed candidates instead of integrating a flow:

$$ \min_\phi\; \mathbb{E}_{\text{offline}}\!\left[\frac{1}{H}\sum_{h=1}^{H}\operatorname{CE}\!\Big(p^{\text{tgt}}(G_h),\,\ell_\phi(s_t,a_{t:t+h})\Big)\right],\qquad G_h = r^{(h)}_t+\gamma^h V^\star(s_{t+h}), $$

with \(V^\star(s_{t+h})\) the expected-prefix-max over the precomputed candidates at \(s_{t+h}\).

Assumed per-frame data schema (to confirm with the data team)

latent      float32 [2048]       # RL-token: 3 cams + proprio fused (state input, latent-only)
reward      float32 []           # per-frame reward (sparse: -1/step, 0 success, -C_fail failure; normalized)
action      float32 [14]         # executed behavior action at this frame -> builds a_{t:t+H}  (standard LeRobot field)
done        bool/float []        # episode-end flag

(+ episode_index / frame_index)

Per your note, each HF transition carries latent, reward, and the 32 vla_chunks; we additionally rely on the standard LeRobot action (the executed behavior chunk) for the prediction side — confirm it is retained.

Value support \([v_{\min},v_{\max}]\) must cover the realized return range. HL-Gauss puts zero mass outside the support, so if returns can fall below \(v_{\min}\) (or above \(v_{\max}\)) the critic cannot represent them. We estimate \([v_{\min},v_{\max}]\) from the dataset's discounted returns (a small setup script) once the reward column is available.
Bootstrap N vs precomputed 32. We precompute 32 chunks/state (deploy can use best-of-32), but the training bootstrap uses num_action_samples (default 8) of them — the ACSAC ablation shows \(N\ge6\) saturates, and full 32 at \(H{=}50\) is ~4× slower. Configurable.
세팅 차이: 논문은 flow-BC와 critic을 온라인에서 함께 학습하지만, 우리는 frozen VLA로 critic만 offline 학습합니다. 상태는 latent(2048)만 사용합니다(proprio가 이제 VAE에 들어가 latent에 포함됨), 행동은 14-D, \(H{=}50\). bootstrap 후보 chunk는 학습 중 VLA를 호출하지 않고 미리 계산해 저장한 것을 사용합니다. 보상은 성공 \(-1,\dots,-1,0\) / 실패 \(-1,\dots,-1,-C_{\text{fail}}\)이며 정규화 후 value가 \([-1,0]\)에 들어가므로 \(v_{\min}=-1,\,v_{\max}=0\). support는 실제 return 범위를 반드시 덮어야 합니다(§7 히스토그램 확인).

§7 · Verification ★

How to convince yourself the distributional critic is correct. ✅ = already run during implementation; ☐ = to run on real data / hardware.

CheckExpectation
✅HL-Gauss round-trip: scalar \(y\to p^{\text{tgt}}(y)\to\sum_i p_i c_i\)recovers \(y\) (we saw \(-9.3,-5,-1.2\to\) same); \(\sum_i p_i=1\)
✅Shapes at real sizes (\(H{=}50\), latent 2048, \(M{=}101\))logits \((K,B,50,101)\); expected value \((K,B,50)\); CE \((K,B,50)\)
✅Both types train end-to-end (create→update→extract)finite losses; regression bit-identical to before; distributional \(q\_mean\) inside \([v_{\min},v_{\max}]\)
✅Prefix consistency (causal mask) — inherited from the backbonechanging \(a_{h:H}\) leaves \(Q(\cdot,a_{1:h})\) bit-unchanged
☐Support coverage: histogram of dataset returns vs \([v_{\min},v_{\max}]\)<1% mass clipped at the edges
☐Calibration: binned predicted \(\hat Q\) vs realized Monte-Carlo return \(\hat G\)points track the diagonal (ACSAC Fig. 4)
☐Distributional ≈ regression control on the same dataexpected values agree within noise; distributional usually steadier

The quickest "is it learning?" signal: watch critic/q_mean and critic/v_next_mean stay inside \([v_{\min},v_{\max}]\) and converge, and the predicted-distribution entropy shrink over training (peaked histograms = confident value). If \(q\_mean\) pins to an edge, widen the support.
