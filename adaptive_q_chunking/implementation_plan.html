Adaptive Q-Chunking — Implementation Plan

Faithful integration of ACSAC: Adaptive Chunk Size Actor-Critic with Causal Transformer Q-Network (Chen et al., 2026, acsac.pdf) into the existing QC / Q-Chunking codebase. Scope: the critic-learning component of the RSS 2026 Post-Finetuning Robot Foundation Model Challenge.

RL criticJAX / FlaxOGBench offline + offline-to-online

One-paragraph summary. ACSAC replaces QC's fixed-chunk MLP critic Q(s, a1:H) with a causal Transformer that emits, in a single forward pass, the Q-values of every prefix Q(s, a1:h) for h = 1…H. The critic is trained with a per-horizon multi-step TD loss (one target per prefix length, averaged at the gradient level), and the policy is the QC flow-BC policy extracted by joint rejection sampling over (candidate index n, prefix length h): at each replanning state we sample N length-H chunks, score all N×H prefixes, execute the arg-max prefix a(n*)1:h*, and replan at the next chunk boundary. The chunk size therefore becomes state-dependent instead of fixed.

1. Paper Summary

Main idea

Long-horizon, sparse-reward tasks make single-step TD learning slow (bootstrapping error accumulates over many Bellman updates). Action chunking — running RL over action sequences at:t+H — gives multi-step value backups without the off-policy bias of n-step returns, because the critic conditions on the exact executed sequence. But existing chunking methods (QC, QC-FQL, DEAS, DQC, MAC) use a fixed H, which forces one global trade-off between reactivity (short chunks, frequent replanning) and temporal coherence (long chunks, fast backups). The optimal trade-off is state-dependent. ACSAC makes the execution horizon adaptive by learning prefix-conditioned Q-values on a shared return scale and selecting the horizon that maximises the critic at each replanning state.

Network architecture (§4.1, App. B)

  • Causal Transformer critic. Ingests the token sequence (st, at, at+1, …, at+H-1) (1 state token + H action tokens) and outputs H heads [Q(s,at:t+1), …, Q(s,at:t+H)] in one pass.
  • A causal attention mask makes the h-th head depend only on (s, at:t+h) ⇒ prefix consistency (Prop. G.7): no causal leakage from future actions, unlike an MLP fed truncated chunks.
  • Pre-LayerNorm before every attention and feed-forward sublayer. LayerNorm bounds the critic's NTK (SEEM [43]), which is what lets ACSAC use the online critic with stopped gradient as the bootstrap target (no separate Polyak target network needed).
  • Size (all tasks): n_layer=2, n_head=8, d_head=16 ⇒ n_embd=128. Critic ensemble K=2; ACSAC takes the min of the two Q-values for both the bootstrap target and policy extraction. Smallest parameter count of all baselines (2.46M) precisely because one shallow shared backbone serves all prefix lengths.
  • Flow-BC policy (unchanged from QC): a state-conditioned velocity field vθ(u, s, az) over RHd, trained by flow matching on full-length chunks; chunks generated by F=10 Euler steps.

Loss functions

Critic (multi-step TD over all prefixes, Eq. 6):
L(φ) = E(st:t+H+1, at:t+H, rt:t+H)~D [ (1/H) ∑h=1…H ( Qφ(st, at:t+h) − Gh )2 ]
Per-horizon target (Eq. 5): Gh = ∑τ=0…h-1 γτ rt+τ + γh Q̄φ(st+h, π*(st+h))
Flow-BC actor (Eq. 7): L(θ) = E[ ‖ vθ(u, st, az) − (at:t+H − z) ‖2 ], az=(1−u)z+u·at:t+H

Averaging the H squared losses at the gradient level (not the target level) reduces update variance (App. G.5, Lemma G.18) while preserving sparse-reward signal in each per-horizon target.

Target computation — expected-prefix-max backup (§4.2, Def. G.11)

The bootstrap value Q̄(st+h, π*(st+h)) generalises EMaQ's expected-max operator: at the next state st+h sample N length-H chunks from the flow-BC policy, score all N×H prefixes, and take the max:

V*(s') = maxn∈[N], k∈[H] minj∈[K] Qφ,j(s', ã(n)s':s'+k)

Appendix G proves this backup BN,Hθ is a γ-contraction in sup-norm whose unique fixed point is Qπ*, the action-value function of the deployed adaptive-prefix policy. Prefix values across h are cross-horizon comparable (Thm. G.9): all share the unit "total discounted return from st", so the joint arg-max is principled.

Inference / policy extraction (§4.3, Eq. 8)

z(n) ~ N(0, IHd); a(n)t:t+H = πθ(st, z(n)); (n*, h*) = argmaxn∈[N], h∈[H] Qφ(st, a(n)t:t+h); π*(st) = a(n*)t:t+h*

The same extraction rule is used for (a) the bootstrap action in the TD target and (b) deployment. Online fine-tuning (Alg. 1): act with the selected prefix open-loop, replan at the chunk boundary, append single-step transitions to D, keep training with the same offline objective (no special O2O tricks).

2. Comparison with Existing QC

Similarities (reused wholesale)

  • Flow-BC policy = ActorVectorField + flow-matching loss + Euler compute_flow_actions — identical to QC (acfql.py, actor_type="best-of-n").
  • Rejection-sampling ("best-of-N") extraction concept — ACSAC adds a second axis (prefix length).
  • Dataset chunking: Dataset.sample_sequence already returns per-step cumulative discounted rewards, all intermediate next-observations, masks and a valid flag — exactly the quantities ACSAC's per-horizon targets need. No dataset change required.
  • Train/eval scaffolding: main.py (offline + online), evaluation.py (action-queue rollout), flax_utils (ModuleDict/TrainState), config-via-get_config(), W&B/CSV logging, replay buffer, OGBench env utils.

Potential implementation challenges

  • Prefix consistency must come from architecture, not data (Remark G.8) ⇒ correct causal mask + read each prefix-Q from the matching action token's hidden state.
  • Variable-length output at deploy breaks jax.jit (output shape depends on data). Solved by returning the full best chunk + a scalar h*; the Python rollout loop slices the first h* actions.
  • Cost of the bootstrap: per transition we bootstrap at H next-states × N proposals, each an F-step flow rollout. Batched as one (B·H·N) tensor; comparable to QC's N=32 sampling.
  • Shared online/target critic stability hinges on Pre-LayerNorm being correct (SEEM). Pre-LN + final LN included; Polyak target available as a fallback flag.

New modules

  AQCAgent(flax.struct.PyTreeNode)

Token layout & prefix read-out

tokens:   [ s_t | a_t | a_{t+1} | ... | a_{t+H-1} ]      length L = H+1
position:   0     1      2              H
causal mask: token i attends to j ≤ i
read head h from action token at position h  ->  Q(s_t, a_{t:t+h})   (h = 1..H)
  token 1 sees {s_t, a_t}                 -> Q(s, a_{t:t+1})
  token H sees {s_t, a_t,...,a_{t+H-1}}   -> Q(s, a_{t:t+H})

Data flow (one critic update)

batch = sample_sequence(B, H, γ)
  observations      (B, obs)            = s_t
  actions           (B, H, act)         = a_{t:t+H}
  rewards           (B, H)              = cumulative discounted r^h_t  (index h-1)
  next_observations (B, H, obs)         = s_{t+1..t+H}
  masks, valid      (B, H)

prediction:  q = critic(s_t, flatten(a))             -> (K, B, H)     # all prefixes, one pass
target:      V* = _target_values(next_obs (B*H,obs)) -> (B, H)        # expected-prefix-max
             G_h = rewards[:,h] + (γ^h)*masks[:,h]*V*[:,h]          # per horizon
loss:        mean_h mean_K ( valid[:,h] * (q - stop_grad(G_h))^2 )

4. Training Pipeline

Data loading

Unchanged from QC. --horizon_length=H sets the max chunk size; sample_sequence(B, H, γ) returns the full sub-trajectory with per-step cumulative rewards/masks and a per-step valid mask. --sparse reproduces scene/puzzle sparse rewards; large datasets (cube-quadruple-100M) are streamed via --ogbench_dataset_dir + --dataset_replace_interval exactly as in QC.

Critic learning

Forward the Transformer once on (s_t, a_{t:t+H}) to get all H prefix-Q predictions; regress each to its per-horizon target; average the H squared errors. Both ensemble members regress to the same (min-aggregated) target. valid[:,h] masks horizons that cross an episode boundary.

Target computation

Expected-prefix-max (Eq. 19): at each of the H next-states sample N flow-BC chunks, score the N×H prefixes with the critic, take min over K then max over (n,k). Default bootstrap = online critic with stop_gradient (paper); a Polyak target_critic path is available via use_target_critic=True.

Chunk sampling strategy (the adaptive part)

At deployment/bootstrap, draw N length-H chunks, evaluate all prefixes, take the joint arg-max (n*, h*). h* is logged per step so the executed-chunk-size distribution (paper Fig. 3) falls out for free. A adaptive_chunking=False switch forces h*=H to recover a fixed-chunk Transformer-critic ablation (the paper's QT-QC-style control).

Evaluation procedure

Greedy adaptive rollout: sample_actions returns (full_chunk, h*); the rollout loop executes the first h* actions, then replans. 50 episodes; success rate + mean/standard-deviation of h* reported. Offline protocol = average of last 3 eval epochs (800k/900k/1M); O2O = success at 1M and 2M.

6. Risks, Open Questions & Design Decisions

DD-1 — Bootstrap target = online critic with stop-gradient (paper-faithful default). App. B states ACSAC bootstraps from the online critic with the gradient stopped, relying on Pre-LayerNorm (SEEM) for stability; the τ=5e-3 in Table 2 is "(when used)" for the target-network baselines. Decision: default use_target_critic=False (faithful). Impact / mitigation: if a run shows value divergence, set use_target_critic=True for a Polyak target (machinery included). Pre-LN + final LayerNorm are always on.
DD-2 — One output head per prefix position (paper-faithful). Prop. G.7 states "one output head per position". Decision: per_position_head=True (default) — a distinct (n_embd→num_atoms) projection at each of the H action positions, implemented as a single batched einsum. Impact: each prefix length gets its own readout while sharing the Transformer backbone; a tied/shared head remains available via per_position_head=False for ablation. (Earlier drafts used a shared head for parameter economy; switched to per-position to match the paper.)
DD-2b — Distributional critic (HL-Gauss), selectable. Beyond the scalar regression critic, a categorical value head is available via critic_type=distributional (HL-Gauss: scalar target → truncated-Gaussian soft histogram over num_atoms bins, cross-entropy loss, expected value = Σ pici). The per-prefix categorical naturally fits the per-position heads (DD-2). Value support (v_min/v_max) is set principledly via --support_type: data (p1/p99 of dataset return-to-go + margin), universal (r/(1-γ) bounds), or custom. Decision: default regression (matches the paper's critic); distributional is opt-in for the VLA/offline setting where a bounded return scale helps. See aqc_distributional_guide.html for the full derivation.
DD-3 — Bootstrap at all H next-states (no horizon subsampling). Eq. 6 sums targets over h=1…H, each bootstrapping at st+h. Decision: compute all H (faithful), batched into one (B·H·N) critic call. Impact: dominant compute cost; a documented target_horizon_subsample hook is left for future speed-ups but defaults to full.
DD-4 — Config lives in aqc.py:get_config(), not a YAML. The QC codebase configures agents through ml_collections + config_flags; an external aqc.yaml would fork the config system. Decision: keep the native pattern and ship example .sh launch scripts as the "example configuration". Impact: consistent with QC, switchable via --agent agents/aqc.py --agent.* ….
OQ-1 — Positional embeddings. The paper does not specify the positional scheme. Assumption: learned absolute positional embeddings over L=H+1 tokens (standard; H is small). Revisit if cross-horizon calibration (Fig. 4) looks off.
OQ-2 — Transformer MLP ratio & activation. Not given. Assumption: GELU, MLP hidden = 4·n_embd = 512 (coincides with the shared MLP-width hyperparameter), matching QC's nn.gelu MLPs.
OQ-3 — Pixel observations. Encoder plumbing is wired (encoder output becomes the state token) but AQC is validated on state-based OGBench. Pixel AQC is left as documented future work.
Backward compatibility. No QC behaviour changes: same files run the same way; the eval/online loops branch only on a tuple return, which QC agents never produce.

7. How to Run (OGBench)

Run from adaptive_q_chunking/qc/. --horizon_length is the max chunk size H; --agent.num_action_samples is the rejection-sampling size N.

Offline + offline-to-online (cube-double, H=5, N=4)

MUJOCO_GL=egl python main.py \
  --agent agents/aqc.py \
  --run_group=aqc --env_name=cube-double-play-singletask-task1-v0 \
  --horizon_length=5 --agent.num_action_samples=4 \
  --offline_steps=1000000 --online_steps=1000000

Sparse domains (scene / puzzle add --sparse=True)

MUJOCO_GL=egl python main.py --agent agents/aqc.py \
  --env_name=scene-play-singletask-task1-v0 --sparse=True \
  --horizon_length=5 --agent.num_action_samples=4

Offline-only smoke test

MUJOCO_GL=egl python main.py --agent agents/aqc.py \
  --env_name=cube-double-play-singletask-task1-v0 \
  --horizon_length=5 --agent.num_action_samples=4 \
  --offline_steps=50000 --online_steps=0 --eval_episodes=10

Evaluation runs automatically every --eval_interval steps inside main.py and reports success rate and the adaptive chunk-size distribution. See AQC.md for the full file map and the QC⇔AQC switch.
