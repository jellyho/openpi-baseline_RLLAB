VLA frozen-policy offline critic 학습 시작 전, 관련된 모든 부분을 한 번에 검토하는 checklist. 끝나면 진짜 실험 시작. 코드: vla_aqc.py · vla_data.py · vla_config.py · utils/transformer.py · vla_train.py.

1. 데이터 (Data)

상태항목현재 상태
DONE데이터 구성 파악teleop 831 (expert, 0 fail) + inference 288 (policy, 124 fail) = 1119 episodes / 3.08M frames. (원본은 expert/failure/success+hil 3 subset.)
DONEfailure reward label 확인inference episode 288개 중 124개에 terminal reward −0.5 정상 존재 (원본 failure-data 125개와 일치). 초반에 "4개뿐"이라고 잘못 봤던 건 한 파일의 row-group 하나만 스캔했기 때문 — episode가 여러 파일/RG에 분산되어 전체를 합쳐야 정확. check_dataset.py로 직접 확인 가능.
DONE전체 데이터 다운로드HF가 4개 큰 파일(~181GB)로 제공, 로컬에 다 있음. (1119 ep 전부 포함.)
DECIDEcommander_filter현재 None (teleop+inference 둘 다 사용). teleop은 mc_return이 전부 −0.02로 동일(전부 성공) → critic 학습 signal이 거의 없음. inference만 쓸지 결정 필요. ← 중요
VERIFY teleop의 학습 가치. teleop 831개는 전부 mc_return≈−0.02 (길이만 다름). distributional critic 입장에서 "전부 같은 값"이라 informative하지 않을 수 있음. 반대로 in-distribution coverage를 넓혀 critic을 안정화시킬 수도 있음. 실험적으로 commander_filter=("inference",) vs None 비교 가치 있음.

2. Reward & Value Support

상태항목현재 값
DONEreward 구조 파악−1e-4/step (living) + −0.5 failure terminal; success → 0. range [−0.5, 0]. (RECAP 방식: −1/T_max, C_fail=T_max/2, T_max≈10000.)
DONEmc_return 검증precomputed γ=0.995, range [−0.5, 0]. recurrence mc_t = r_t + 0.995·mc_{t+1} 수치 확인됨.
DONEvalue supportv_min=−1.0, v_max=0.0 (요청대로). 실제 range [−0.5,0]이 안에 들어가서 OOB=0% 보장. TD critic이 더 낮은 값 학습할 여유.
DONEnum_atoms201 (RECAP과 동일). [−1,0]에서 bin width = 0.005.
DONEHL-Gauss sigma0.75 × (v_max−v_min)/atoms = 0.00375. (Stop-Regressing 권장값.)
VERIFYbin 활용도성공 mass가 [−0.02,0]에 몰릴 수 있음. 학습 후 dist_edge_mass / dist_target_oob 로그 보고 너무 낭비면 좁히기 (예: −0.6). 일단 [−1,0]로 시작 확정.

3. Transformer Architecture

상태항목현재 값
DONEcritic = causal Transformerstate token 1 + macro-action tokens. per-position head (Prop G.7), Pre-LN, 2-ensemble min.
DONEcapacityn_embd=384 (8×48), 4 layers, mlp=1024 → ~13.5M. (state encoder = linear, ACSAC-faithful.)
DONEmacro-action groupingmacro_group_size=10: 50 action → 5 macro-token, sequence L=6. 검증: prefix-consistency ✅, shape (K,B,5,201) ✅. attention 72× / bootstrap 10× 감소.
DECIDEstate encoder MLP 여부기본 linear (2048→384). vla_..._stateenc = MLP(512). 2048→384가 5× 압축이라 MLP가 도울 수 있음 → ablation 대상.
DECIDEcapacity sweepOGBench에서 distributional 이득은 critic 클수록 커짐(Stop-Regressing). small(5M)/default(13.5M)/large(35M) 비교 가치.

4. Critic Learning — TD (핵심)

TD critic 학습으로 결정. MC return은 검증용.

Target 공식 (per macro-prefix h)

G_h = Σ_{j<h} γ^j · r_{t+j}  +  γ^h · valid_h · V*(s_{t+h})

V*(s') = min_K  max_{n∈[32], h'∈[H]}  Q_φ̄(s', base_action(s')^{(n)}_{1:h'})    ← 32 candidate joint-max

terminal(−0.5 frame)에서는 V*(s') = mc_return(s')   ← max 대신 penalty 보존
상태항목현재 상태
DONEper-prefix multi-step TD각 prefix h={10,20,30,40,50}마다 target. critic_loss_td.
DONEaction-chunk reward sumcum_reward[h] = Σ_{j<h} γ^j·r[i+j], step 단위 discount (γ=0.995). loader에서 사전 계산. env 불필요 — reward column에서 직접 합산.
DONEbootstrap = 32-candidate joint-max_expected_prefix_max: base_action 32개 × prefix max, ensemble min. tiled(메모리 안전).
DONEterminal 처리term flag → bootstrap을 mc_return으로 대체 (−0.5 penalty 보존). terminal_uses_mc=True.
DONEepisode boundary maskingvalid flag로 episode 넘어가는 prefix 제외.
DONEno target networkonline critic + stop_gradient (논문 방식). target_critic 코드 완전 제거함. ← 사용자가 요청한 대로.
DONEdiscount γ0.995 (TDConfig.discount). cum_reward·bootstrap 모두 step 단위 γ.
DONEdistributional lossper-prefix HL-Gauss cross-entropy. target → soft histogram → CE.
VERIFYterminal mc_return 스케일mc_gamma=1.0(undiscounted)인데 TD는 γ=0.995. 다만 terminal에서는 둘 다 −0.5로 일치(terminal이 episode 마지막이라 future 없음) → 실질 문제 없음. 한번 눈으로 확인 권장.
VERIFYexecuted chunk vs base_action현재 critic의 input은 executed action[t:t+H], bootstrap의 후보는 base_action. 데이터팀에 "critic이 어느 action에 conditioning돼야 하는지" 확인하면 더 확실.

5. MC Return — 검증용

상태항목현재 상태
DONEMC regression 경로 존재target_kind='mc' + critic_loss_mc. 필요시 baseline으로 사용 가능.
DECIDE검증 방법학습된 TD Q 와 mc_return 비교: TD Q ≥ mc_return 이면 policy improvement 확인. eval에서 q_vs_mc_corr 로깅 중 — TD value > MC value 인지 추가 지표 넣을지 결정.
DECIDEundiscounted vs precomputedmc_gamma: 1.0(undiscounted) / None(precomputed γ=0.995). TD가 메인이라 당장은 무관하지만, MC baseline 돌릴 때 결정.

8. 아직 결정/확인해야 할 것 (요약)

꼭 결정/확인할 것 (실험 전):
  1. DECIDE commander_filter — teleop 포함 vs inference만. (teleop은 mc 전부 −0.02라 signal 약함.) ← 가장 중요
  2. VERIFY executed chunk vs base_action conditioning — 데이터팀 확인 시 더 확실.
  3. VERIFY GPU throughput — --timing_steps 100로 it/s 측정 → num_train_steps·batch_size 확정.
  4. DECIDE MC baseline 먼저 vs TD 바로 — 안정성 위해 MC sanity 먼저 권장하나, TD가 메인.
나중에 ablation으로 봐도 되는 것:
  • DECIDE num_atoms (201 vs 51), state_encoder (linear vs MLP), capacity (5/13.5/35M), v_min (−1 vs −0.6), mc_gamma (1.0 vs 0.995).
  • DECIDE QGF test-time guidance — critic 잘 학습된 후, openpi/models/pi0.py:sample_actions의 step 함수에 gradient guidance 추가 (별도 논의됨).
