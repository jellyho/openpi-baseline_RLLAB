HL-Gauss distributional head

The critic outputs \(K \times \text{macro\_H} \times \text{num\_atoms}\) logits per forward pass. A scalar target \(y\) is soft-histogrammed via a truncated Gaussian CDF: \[p_i = \frac{\Phi\!\left(\tfrac{c_{i+1}-y}{\sqrt{2}\sigma}\right)-\Phi\!\left(\tfrac{c_i-y}{\sqrt{2}\sigma}\right)}{\Phi\!\left(\tfrac{c_K-y}{\sqrt{2}\sigma}\right)-\Phi\!\left(\tfrac{c_0-y}{\sqrt{2}\sigma}\right)}, \qquad \hat{V} = \sum_i p_i c_i, \qquad \sigma = 0.75\,\frac{v_{max}-v_{min}}{N_{atoms}}\]

num_atoms — current: 201 updated

Changed from 51 to 201 to match RECAP (which uses 201 bins). With [−1, 0] support: bin width = 0.005 — resolves value differences ≥ 0.5%. The OGBench ablation (a51 ≫ a201 in small 0.94M critic) was likely a small-critic artefact; the VLA critic is 13.5M and the dataset is 3× larger, making 201 bins tractable and preferable.

Why undiscounted (RECAP Eq. 1)

RECAP trains \(p_\phi(V | o_t, \ell)\) by minimizing \(H(R^B_t(\tau),\, p_\phi)\) where \(R_t(\tau) = \sum_{t'=t}^T r_{t'}\) — no discount factor. Key reasons:

  • Simpler, more stable — no discount horizon to tune; the target is the actual total cost incurred.
  • Consistent across episode lengths — discounted returns conflate "short failure" with "slow success"; undiscounted makes them clearly distinct.
  • Wider, more informative range — verified on real data: undiscounted [−0.10, −0.007] vs precomputed [−0.02, −0.006] in a clean-episode row group; undiscounted better separates episode quality via accumulated living cost.
Implementation: VLALeRobotDataset(mc_gamma=1.0) runs a single backward pass over each episode in each row-group (_compute_mc), accumulating running += rew[i] and resetting at episode boundaries. Cost: negligible (pure numpy loop over rewards, no parquet re-read). The value_support property auto-returns (-1.0, 0.0) when mc_gamma=1.0.

Efficiency gains from macro-grouping:
  • Attention matrix: 51×51 = 2601 → 6×6 = 36 entries per head (72× reduction)
  • Bootstrap forward calls: 32×50 = 1600 → 32×5 = 160 per step (10× reduction)
  • Prefix candidates: 50 → 5 (all meaningful: 10/20/30/40/50 real steps = 167ms granularity)

5. CAL-QL Discussion

What it is

CAL-QL (Nakamoto et al. 2023) adds an offline conservatism penalty: \(\mathcal{L}_{CQL} = \alpha\!\left(\mathbb{E}_{a\sim\pi}[Q(s,a)] - \mathbb{E}_{a\sim\mathcal{D}}[Q(s,a)]\right)\), making the critic pessimistic on OOD actions.

Do we need it?

ingredientavailable?how
MC supervision (in-distribution)✅undiscounted MC from raw rewards
Policy Q estimate \(\mathbb{E}_{a\sim\pi}[Q]\)✅base_action = 32 policy samples/state
Dataset Q estimate \(\mathbb{E}_{a\sim\mathcal{D}}[Q]\)✅action = executed behavior action
Penalty weight \(\alpha\)needs tuningextra hyperparameter
Verdict: defer until needed. The MC regression + precomputed in-distribution base_action candidates keep the critic in-distribution by construction. CAL-QL matters when the critic overestimates OOD actions at rollout, which we don't yet know is a problem.

If needed, the one-line addition to critic_loss_mc in vla_aqc.py: 
L_cal = alpha * (Q(s, base_action).mean() - Q(s, executed_action))
L_total = L_mc + L_cal

6. Experiment Plan

  1. MC baseline — vla_mc_macro: undiscounted MC (γ=1), 201 atoms, [−1,0] support, 5 macro-prefixes. This is the RECAP-equivalent. Eval: Q̂ vs mc_return correlation, prefix_spread > 0.
  2. AQC-TD — vla_aqc_td_macro: same config + 32-candidate joint-max bootstrap. Switch once MC loss converges.

  4. CAL-QL — only if Q diverges or critic over-selects bad prefixes.
