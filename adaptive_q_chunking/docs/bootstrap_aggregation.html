Bootstrap Target Aggregation: BoN(max) vs Softmax

VLA AQC critic의 TD bootstrap target \(V(s')\)을 만들 때 후보×prefix Q들을 어떻게 하나로 합치는가. 현재는 hard max(Best-of-N), conservatism을 위해 softmax/mellowmax로 바꾸는 안을 검토. ACH 논문(q_softmax.pdf)과 비교.

0. 어디 얘기인가 — _expected_prefix_max

TD target은 \(\ \text{target} = R_{1:h} + \gamma^{h}\,V(s')\ \) 이고, 여기서 \(V(s')\)을 만드는 게 vla_aqc.py의 _expected_prefix_max. 핵심은 마지막 한 줄:

qs = self._prefix_values(params, states, chunks)   # (K, M*N, macro_H)
q  = qs.min(axis=0)                                 # ensemble-min -> (M*N, macro_H)
return q.reshape(M, N * macro_H).max(axis=-1)       # (M,)  ← 이 max 가 논점

각 next-state \(s'\)마다 N=32 후보 × macro_H=5 prefix = 160개 scalar Q가 있고, 그 중 하나의 값으로 \(V(s')\)을 정합니다.

1. softmax가 두 개라 헷갈림 — 먼저 구분

softmax ① (atoms)softmax ② (aggregation, 제안)
위치_prefix_values 안.max(-1) 자리
무엇에201개 atom (value bin)160개 Q값
목적분포 → scalar Q 1개160개 Q → \(V(s')\) 1개로 집계
온도 β없음β로 max↔mean 조절
logits (…, 201 atoms) │ softmax①(atoms) + from_probs ← 이미 있음: 분포 → scalar Q ▼ 160개 scalar Q (32 후보 × 5 prefix) │ ← 여기! max(-1) 를 softmax②(Q값들) 로 교체하는 안 ▼ V(s') (M,)
제안하는 softmax는 ②번 — Q값을 다 구한 후, max 직전에 그 Q값들 위에 적용. logit/atom 단계의 softmax①과는 완전히 별개.

2. 현재: max(BoN)가 의미하는 것 current

$$ V(s') = \max_{n\in[N],\, h'\in[H]}\ Q_{\bar\phi}(s', a^{(n)}_{1:h'}) $$

이건 Bellman optimality backup (\(\max_{a'}Q\))을 두 축에서 동시에 하는 것:

  • 후보 축 max = 행동 선택(policy improvement): frozen VLA의 32개 후보 중 best를 골라 \(\max_a Q\)를 근사 → Best-of-N / EMaQ / BFN.
  • prefix 축 max = chunk length 선택: 10~50 step 중 가치 최고 길이 선택.
문제: optimistic bias. Q에 추정 노이즈가 있으면 160개 중 max는 양의 노이즈 꼬리를 집음 → 과대평가. offline에서 value가 \(v_{max}=0\)로 쏠리는 원인. (actor 없이 후보 max로 improvement하는 구조라 더 그럼.)

3. soft 대체 — 원리 option

\(q_f = q.\text{reshape}(M, 160)\)일 때, max를 다음으로 교체:

(A) Softmax-weighted mean (Boltzmann, ACH 방식)

$$ V = \sum_j w_j q_j, \qquad w_j = \frac{e^{\beta q_j}}{\sum_k e^{\beta q_k}} $$ 
V = (jax.nn.softmax(beta * qf, axis=-1) * qf).sum(-1)

(B) Mellowmax (log-sum-exp, contraction 보장)

$$ \text{mm}_\beta(q) = \frac{1}{\beta}\log\!\Big(\frac{1}{J}\sum_j e^{\beta q_j}\Big) $$ 
V = jax.scipy.special.logsumexp(beta*qf, -1)/beta - jnp.log(qf.shape[-1])/beta
원리 — "soft 정책의 가치". max는 "다음 state에서 무조건 best 하나 고름"(greedy). softmax 기대값은 "\(\pi(j)\propto e^{\beta q_j}\)인 확률적 정책으로 고를 때의 평균 가치". 이게 soft Q-learning / max-entropy RL의 backup이고, ACH의 \(V=\mathbb{E}_{\ell\sim\pi_L}[Q]\)와 같은 원리.

둘 다 보간이며 항상 \(\ \text{mean}(q)\le V_\beta \le \max(q)\ \), β에 대해 단조증가:

β분포V
\(\to\infty\)argmax에 확률 1= max (hard max 복원, 낙관)
중간Q 높은 쪽 가중max와 mean 사이
\(\to 0\)균등= mean (160개 평균, 보수)
왜 conservative. 가중평균이라 항상 max 이하 + 운 좋은 outlier 하나를 나머지 159개로 희석 → overestimation 완화. target도 argmax가 후보 간에 튀지 않고 매끄럽게 변해 학습 안정.

4. β = inverse temperature (역온도)

β는 exp 안에서 Q에 곱해져 분포의 뾰족함을 조절. softmax는 shift-invariant라 절댓값이 아니라 Q 차이만 중요하고, 실제 peakedness는 \(\ \beta\times(q_i-q_j)\ \)로 결정됨.

숫자 예시 (두 후보 \(q=[-0.2,-0.4]\), 차이 0.2)

ββ·차이weightsV해석
→00[0.50, 0.50]−0.30mean
10.2[0.55, 0.45]−0.29거의 mean
51.0[0.73, 0.27]−0.25약하게 max쪽
204.0[0.98, 0.02]−0.20≈ max
→∞∞[1, 0]−0.20max
핵심 — β는 Q차이 스케일에 맞춰야. 우리 Q차이가 ~0.1–0.3이면 β=1은 거의 mean. 의미 있는 soft-max를 하려면 β≈10–50 sweep 필요. (절댓값 \([-1,0]\)은 무관, 차이만 영향)

5. ACH 논문의 softmax/β q_softmax.pdf

ACH는 우리와 다른 구조: 별도 \(V(s)\) head를 두고 Q를 V로 bootstrap, V-target은 chunk length에 대한 softmax.

$$ \pi_L(\ell) = \frac{\exp(\beta\, Q_\phi(s_t, a'_{t:t+\ell}))}{\sum_{n}\exp(\beta\, Q_\phi(s_t, a'_{t:t+n}))} \qquad V_\phi(s_t)\leftarrow \mathbb{E}_{\ell\sim\pi_L}[Q_{\bar\phi}(s_t, a'_{t:t+\ell})] $$
ACH우리(AQC)
softmax 대상chunk length \(\ell\) (10개)후보×prefix (160개)
action 후보단일 policy 샘플 a' (actor가 개선)frozen VLA 32 후보 (BoN으로 개선)
bootstrap별도 V(s) headQ→Q 직접 (V head 없음)
β 값1 고정미정 (검토중)
β 주목적online 탐험 분포 모양offline conservatism
ACH의 β=1은 우리 β=1과 다르다. ACH Q차이 ~0.4(Fig.2) / \(|Q|\approx194\) → β=1이면 꽤 평평. 근데 ACH엔 OK: (a) 탐험 목적이라 평평한 게 좋고, (b) eval은 greedy(argmax)라 β 무관, (c) 길이별 Q가 다 모여있어 V가 β에 둔감. → ACH에서 β의 역할은 value 보수성이 아니라 탐험 분포 모양.

6. 우리의 default 결정

β=1을 default 값으로 까되, default 동작은 max 유지.
β=1은 레퍼런스로 깔끔하지만, 우리 \([-1,0]\) 스케일에선 β=1 ≈ mean = 가장 보수적인 끝 (policy improvement 거의 없는 behavior-평균 bootstrap). "중립값"이 아니라 "강한 보수쪽"임을 인지.
# TDConfig
agg_mode: Literal["max", "softmax", "mellowmax"] = "max"   # 동작 default = 검증된 max
agg_beta: float = 1.0                                      # 값 default = 1 (soft 모드일 때만 효과)
  • default 동작 = max — 지금까지 검증한 것, 안 깨짐.
  • default β = 1.0 — 깔끔한 레퍼런스로 박아둠 (soft 모드 켤 때만 사용).
  • soft 켜면 β sweep (1 → 5 → 10 → 20 → 50). 우리 scale에선 β=1이 거의 mean이라 "보수적이지만 improvement 살아있는" 지점은 β>1일 가능성.
  • terminal은 그대로 mc_return 사용 (집계와 무관).

요약 — 5줄

  1. 논점은 _expected_prefix_max의 마지막 max(-1) 한 줄 (atom softmax①과 무관).
  2. max = Bellman optimality(BoN) → 낙관, offline overestimation 위험.
  3. soft(softmax/mellowmax) = "확률적 soft 정책의 가치" → max↔mean 보간, conservative.
  4. β = 역온도. β×(Q차이)가 효과를 정함 → 우리 \([-1,0]\) scale에선 β=1≈mean, β≈10–50 필요.
  5. default: 동작=max, β=1 박아두고 soft는 ablation으로 β sweep.
