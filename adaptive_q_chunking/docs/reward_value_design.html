Reward & Value Range Design

VLA critic 학습용 reward / value support 설계. 목표: 모든 value가 \([-1, 0]\) 안에 자연스럽게 fit + 평균 길이 성공 trajectory의 시작 value가 −0.3 ~ −0.4 형성. discount \(\gamma = 0.999\) (long-horizon task). MC return은 사용하지 않고 TD bootstrap만 사용.

0. 데이터 사실 (fixed)

항목값

parquet row-group 크기~1,000 행 (episode 하나가 2~3 group에 걸침)
제어 주파수60 Hz
구성teleop 831 (전부 성공) + inference 288 (124 failure)
raw reward 인코딩living = −0.0001, success terminal = 0.0, failure terminal = −0.5

1. Reward 정의 (RECAP-style)

매 step에 작은 living cost, terminal에서 성공/실패 구분:

$$ r_t = \begin{cases} 0 & t = T \text{ 이고 success} \\ -C_{\text{fail}} & t = T \text{ 이고 failure} \\ -r_{\text{step}} & \text{그 외 (평범한 state)} \end{cases} $$

우리가 정해야 할 두 값: 평범 state cost \(r_{\text{step}}\) 와 failure penalty \(C_{\text{fail}}\).

2. Value 정의 — discounted return-to-go

critic이 회귀하는 target은 시작 state 기준 discounted return-to-go:

$$ V(s_0) \;=\; \sum_{k=0}^{T-1} \gamma^{k}\, r_{k} \;+\; \gamma^{T}\, r_{T} $$

이 식을 living 항과 terminal 항으로 분리하면:

$$ V(s_0) \;=\; \underbrace{(-r_{\text{step}})\sum_{k=0}^{T-1}\gamma^{k}}_{\text{living cost}} \;+\; \underbrace{\gamma^{T}\,r_{T}}_{\text{terminal (discounted!)}} $$
핵심 1 — terminal 에도 \(\gamma^T\) discount가 붙는다. failure penalty \(-C_{\text{fail}}\)는 episode 끝(\(T\) step 후)에 있으므로, 시작점에서 보면 \(\gamma^{T}\)만큼 약해진다. 즉 실패가 멀수록(긴 traj) penalty가 작아진다. $$\gamma^{T}\big|_{\gamma=0.999}:\quad \gamma^{100}=0.905,\ \ \gamma^{500}=0.606,\ \ \gamma^{1000}=0.368,\ \ \gamma^{3000}=0.050$$

3. Living cost — geometric sum

등비급수 공식으로 living 항을 정리:

$$ \sum_{k=0}^{T-1}\gamma^{k} \;=\; \frac{1-\gamma^{T}}{1-\gamma} \qquad\Longrightarrow\qquad V_{\text{living}}(s_0) \;=\; -\,r_{\text{step}}\cdot\frac{1-\gamma^{T}}{1-\gamma} $$

\(\dfrac{1-\gamma^{T}}{1-\gamma}\)는 "discount 적용된 유효 step 수". \(\gamma=0.999\)에서 \(\dfrac{1}{1-\gamma}=1000\) 이므로 유효 step 수 \(= (1-\gamma^{T})\times 1000\):

traj 길이 \(T\)\(\gamma^{T}\)유효 step 수 \(\frac{1-\gamma^T}{1-\gamma}\)
1,0000.368632

핵심 2 — effective horizon. \(\gamma=0.999\)는 traj가 아무리 길어도 유효 1,000 step까지만 센다. 그래서 평균 길이(~2,500) traj는 거의 수렴값에 도달.

4. \(r_{\text{step}}\) 결정 — 시작 value가 −0.3~−0.4 되도록

성공 traj는 terminal 항 \(=0\) 이므로 시작 value는 living cost만:

$$ V_{\text{success}}(s_0) \;=\; -\,r_{\text{step}}\cdot\frac{1-\gamma^{T}}{1-\gamma} \;\xrightarrow[\;T\to\infty\;]{}\; -\,\frac{r_{\text{step}}}{1-\gamma} $$

수렴값(가장 음수)을 −0.4로 맞추려면:

$$ -\frac{r_{\text{step}}}{1-\gamma} = -0.4 \;\Longrightarrow\; r_{\text{step}} = 0.4\,(1-\gamma) = 0.4\times 0.001 = \mathbf{0.0004} $$

\(r_{\text{step}} = -0.0004\) 일 때 길이별 시작 value:

평균 길이(~2,500) 성공 traj의 시작 value ≈ −0.35 ~ −0.40 ✅ (raw \(r_{\text{step}}=-0.0001\)이면 수렴값이 −0.1뿐 → too small. loader에서 ×4 relabel.)

5. \(C_{\text{fail}}\) 결정

실패 traj의 시작 value (living + discounted failure):

$$ V_{\text{fail}}(s_0) = -\,r_{\text{step}}\cdot\frac{1-\gamma^{T}}{1-\gamma} \;+\; \gamma^{T}\,(-C_{\text{fail}}) $$
핵심 3 — 두 항은 trade-off다. 실패가 늦으면(\(T\)↑): living은 커지지만(\(\to -0.4\)) failure는 \(\gamma^T\)로 작아짐(\(\to 0\)). 실패가 빠르면(\(T\)↓): living은 작지만(\(\to 0\)) failure는 거의 그대로(\(\to -C_{\text{fail}}\)). → 둘이 동시에 max가 될 수 없어 합이 \(-(r_{\text{step}}/(1-\gamma) + C_{\text{fail}})\)보다 항상 작다.

\(r_{\text{step}}=-0.0004\), \(C_{\text{fail}}=0.5\) 기준 실패 value:

\(C_{\text{fail}}=0.5\) — 즉각 실패 −0.50, 늦은 실패 −0.41 ~ −0.43. 모든 failure value가 −0.5 ~ −0.4 범위 → \([-1,0]\) 안에 안전하게 fit ✅
주의: 늦은 failure ≈ 긴 성공(−0.40)과 value가 거의 같다 — 이는 \(\gamma^T\)가 늦은 penalty를 충분히 감소시키기 때문이며, 어떤 \(C_{\text{fail}}\)을 써도 개선되지 않는 discount의 근본 특성.

6. In-loader Reward Relabel 구현됨

parquet 재작성(181 GB) 없이 loader 안에서 읽은 직후 변환. 두 값의 스케일이 다르므로 따로 처리:

# raw 인코딩: living=-1e-4, success_terminal=0.0, failure_terminal=-0.5
scale = relabel_living / -1e-4          # -4e-4 / -1e-4 = 4.0
out   = rew * scale                     # living -1e-4 → -4e-4, success 0.0 → 0.0
out   = where(rew <= -0.05, relabel_fail, out)   # failure -0.5 → -0.5 (유지)

변환 후 mc_return과 cum_reward는 모두 이 값에서 재계산 → 별도 처리 불필요.

7. False Terminal 버그 수정 버그 수정

발견된 버그 — row-group 경계를 terminal로 오인.
episode 하나(평균 2,500 frame)가 ~1,000행 row-group 2~3개에 걸쳐 있는데, 기존 코드는 end == last_idx_in_group 조건으로 terminal을 판단했다. row-group 경계 frame도 마지막 인덱스이므로 chunk edge 2개 + 진짜 terminal 1개 → false terminal이 53%였다. false terminal 위치의 next_mc_return은 living 값 −0.0004 → critic이 "이 중간 state가 value≈0인 terminal"이라고 잘못 학습.

수정: chunk 위치 대신 reward 값으로 판단. 검증 결과 reward ∈ {0.0, −0.5}는 전체 3.08M frame 중 진짜 episode 끝에서만 등장 (success 368/368, failure 124/124 확인). living −0.0001은 절대 오탐 없음.

$$ \text{is\_terminal}(i) \;=\; \bigl[\,r_i \ge -10^{-6}\bigr] \;\vee\; \bigl[\,r_i \le -0.05\bigr] $$ 
# 기존 (버그)
term = (end == last_idx_in_group) & (valid > 0)   # row-group edge도 terminal로 오인

# 수정
is_term_frame = (rew >= -1e-6) | (rew <= -0.05)  # reward 값으로 판단
term = (valid > 0) & is_term_frame[end_c]          # false terminal 0개 확인
수정 후 검증: false terminal 9 → 0. 남은 terminal은 전부 next_mc_return = −0.5 (진짜 failure). TD bootstrap signal 완전히 정상화. ✅

핵심 요약

  1. terminal에도 \(\gamma^T\) discount → 늦은 failure는 value가 성공과 비슷해진다 (discount의 본질).
  2. 유효 horizon = \(1/(1-\gamma) = 1{,}000\) step → \(r_{\text{step}}=-0.0004\)이면 성공 시작 value ≈ −0.35~−0.40.
  3. living vs failure는 trade-off → 동시에 최대가 될 수 없으므로 \(C_{\text{fail}}=0.5\)면 −1 절대 안 넘는다.
  4. TD bootstrap만 사용, MC return 미사용 → terminal은 reward 값으로 검출 (row-group 경계 오탐 0).
