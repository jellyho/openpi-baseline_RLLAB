Reward & Value Design v3 (현행)

VLA critic 학습용 reward / value 설계. v3 = "raw로 먼저 깔고 → 전역(global) normalize": living −1/step, \(C_{fail}=0.4\,T_{max}\), \(\gamma=0.9999\), 가장 깊은 return \(Z\)로 나눠 \([-1,0]\)에 fit. 데이터셋: *_annotated_v3 (원본 별도 보존). 구버전 v1은 맨 아래 요약.

1. 왜 v3로 바꿨나 (v1의 문제)

v1(γ=0.999, C_fail=−0.5)의 관찰된 문제 — failure 곡선이 평평하다. eval value_curves에서 failure trajectory의 mc_return이 끝까지 거의 −0.4에 붙어 있고 끝에서만 −0.49로 살짝 처짐. 이유: terminal penalty가 \(\gamma^{T-t}\)로 discount되는데 \(\gamma=0.999\)의 유효 horizon은 1000 step뿐 — 평균 2500~5300 step짜리 episode에서는 \(\gamma^{T-t}\approx 0\)이라 시작점에서 failure가 안 보임. 성공/실패 시작값 분리가 ~0.05–0.1로 미미 → critic이 실패를 일찍 구분할 신호가 없음.
v3의 해법 두 가지. (1) \(\gamma=0.9999\) — 유효 horizon 10,000 step ≥ episode 길이 → \(\gamma^{T}\in[0.2,\,1]\), 즉 failure penalty가 episode 시작점에서도 살아있음. value ≈ "정규화된 남은 시간 + 실패 벌점" (RECAP의 steps-to-success 해석과 일치). (2) \(C_{fail}=0.4\,T_{max}\) — 벌점을 living 4,000~6,300 step 분량으로 키워 분리를 명확히.

2. v3 스킴 — 식

Step 1 — raw reward (자연 단위)

$$ r_t^{raw} = \begin{cases} 0 & t=T,\ \text{success}\\[2pt] -C_{fail},\quad C_{fail}=0.4\,T_{max} & t=T,\ \text{failure}\\[2pt] -1 & \text{그 외 (living)} \end{cases} $$

\(T_{max}\) = 그 dataset의 최장 episode 길이 (task별로 다름).

Step 2 — raw return-to-go (γ=0.9999)

$$ G_t \;=\; \sum_{k\ge 0}\gamma^{k}\,r^{raw}_{t+k} \;=\; -\frac{1-\gamma^{\,T-t}}{1-\gamma} \;-\; \gamma^{\,T-t}\,C_{fail}\cdot\mathbb{1}[\text{fail}] $$

\(\gamma=0.9999\Rightarrow \frac{1}{1-\gamma}=10{,}000\): episode 길이(≤ 15,788)에서 \(\gamma^{T-t}\) 가 0.2 밑으로 안 떨어짐 → 벌점이 어디서나 보임.

Step 3 — 전역 normalize

$$ Z \;=\; \big|\min_{\text{dataset}} G\big|,\qquad r_t = \frac{r_t^{raw}}{Z},\qquad \text{mc\_return}_t = \frac{G_t}{Z}\;\in\;[-1,\,0] $$
Z는 파일별이 아니라 dataset 전역. 가장 깊은 return을 가진 episode의 시작점이 정확히 −1에 앉음 (1개 state). normalize는 linear scaling이라 Bellman 일관성 유지 (\(\text{mc} = r + \gamma\,\text{mc}'\)가 나눈 후에도 성립).
