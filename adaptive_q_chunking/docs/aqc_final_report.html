Adaptive Q-Chunking + Distributional Critic — Final Report

OGBench (cube-double) verification of the ACSAC critic and its HL-Gauss distributional variant. Companion docs: implementation_plan · distributional guide · reading guide · VLA critic learning.

요약 (Korean). ACSAC critic와 HL-Gauss distributional critic를 OGBench cube-double에서 검증. 핵심 결과 — (1) 논문 충실 설정(target network 제거 + position마다 output head)이 결정적: regression이 0.43→0.98로 회복. (2) num_atoms는 적을수록 좋았다: task3에서 a51=0.90 ≫ a101=0.37 ≫ a201=0.27 (a51 distributional이 regression 0.62보다도 높음). (3) distributional ≈ regression (task4/5에서 비슷), reward-norm ≈ 이론적 support[-200,0]. (4) sparse에서는 data-centric support만 살아남음 (universal은 chunk가 h=1로 붕괴). 아래에 config·결과·코드 리뷰 위치 정리.

1. Setup

Method. Causal-Transformer prefix critic that scores every prefix \(Q(s,a_{1:h})\) for \(h=1..H\) in one pass; deployment picks the prefix maximizing the critic (adaptive chunk size). The distributional variant replaces the scalar head with an HL-Gauss categorical head over a value support \([v_{min},v_{max}]\) and trains with cross-entropy. See the distributional guide for the math.

Common config (held fixed unless noted): env cube-double (state-based), discount=0.99, batch_size=256, horizon_length H=5, num_action_samples N=4, ensemble K=2 (min-agg), flow-BC actor, use_target_critic=False (paper), per-position output head (paper, Prop G.7), Adam lr=3e-4, offline 500k–1M steps, 30 eval episodes. Env: conda deas_real (jax 0.6), SLURM sweeps.

1.1 Value support \((v_{min}, v_{max})\) — how it was set in every distributional run

The HL-Gauss critic needs a fixed value support. We always set it one of four ways (the exact runtime values are printed per run as [support] ... in the logs):

support mode\(v_{min}, v_{max}\)how derived
data (DEAS-d)per-task; dense cube-double ≈ [−210, 0], sparse ≈ [−100, 0]p1 / p99 of the dataset's discounted return-to-go + 5% margin, estimated at runtime (estimate_value_support)
universal (DEAS-u)\([r_{min}/(1{-}\gamma),\, r_{max}/(1{-}\gamma)]\); sparse (γ=0.99) = [−100, 0]theoretical return bound from the per-step reward range (universal_value_support)
custom [−200, 0]v_min=−200, v_max=0manual = theoretical \(r_{min}/(1{-}\gamma)\) (r_min≈−2, γ=0.99); the _theo runs
reward-norm [−1, 0]v_min=−1, v_max=0rewards scaled so return-to-go lands in [−1,0]; support then fixed (compute_reward_scale)

In every results table below, the support mode is named next to each distributional row; refer here for its exact \((v_{min},v_{max})\). Regression rows have no value support (scalar critic).

Finding 4. On sparse, the universal support (\(r/(1-\gamma)\)) collapses — chunk size falls to ~1.0, i.e. the adaptive selection degenerates to h=1 and success →0 — while the data-centric support survives (0.12). The universal bound is far too wide for sparse returns, so almost all probability mass lands in a couple of bins. On dense they're comparable. Finding 5. γ=0.9995 is the sweet spot for the distributional critic — it peaks both tasks (best overall task5 = dist-data @0.9995 = 0.85, beating every γ=0.99 run). Pushing to γ=0.9999 degrades (task5 0.85→0.60; task4→0): the effective horizon \(1/(1-\gamma)=10\text{k}\) steps makes the value target too long/flat to fit. γ=0.99 (§3.3) and 0.999 are comparable and safe. Regression peaks at the lower γ (0.999). (2 seeds — noisier than the 4-seed γ=0.99 runs; treat ±0.05 as within noise.)

4. What worked / what didn't — synthesis

Worked

  • No target network + per-position head (paper-faithful) — the single biggest factor (§3.1).
  • Regression critic is a strong, reliable baseline: ~0.98 on the easy dense task1, and the overall most consistent across tasks.
  • Distributional with few atoms (51) — can match or beat regression (task3: 0.90 vs 0.62), and is the configuration to carry forward.
  • Data-centric value support — robust on both dense and sparse.
  • Discount γ=0.9995 — sweet spot for the distributional critic (best task5 = 0.85); γ=0.99–0.999 also safe.
  • Dense rewards for cube tasks (matches the paper).

Didn't work / pitfalls

  • Polyak target network — crippled the chunked critic (0.98→~0.6 in early runs). Removed.
  • Many atoms (101/201) — consistently worse than 51 (§3.2).
  • Universal support on sparse — degenerate (chunk→1, success→0) (§3.5).
  • Unnormalized value scale — early distributional runs had q_mean≈-95 because OGBench dense reward isn't in [-1,0]; fixed by estimating the support from data (or normalizing).
  • Sparse cube-double — low success for all methods; dense is the right setting.
  • Discount γ=0.9999 — too high; degrades the value fit (task5 0.85→0.60, task4→0). Effective horizon 10k steps is too long/flat.
  • Distributional is not a free win — on task4/5 it only matched regression; gains are task-dependent.

6. Recommendation & open items

  • Default config: regression OR distributional a51, dense, target=OFF, per-position head, data-centric (or tight fixed) support, H=5.
  • For the VLA phase: carry forward a51 + fixed [-0.5,0] support (the mc_return range); distributional is worth keeping but should earn its place per-task vs regression.
  • Reproduce: sweep configs in scripts/aqc/sweep/cfg_*.json; raw per-run CSVs were in the OGBench working dir (also in W&B project AQC). Numbers here are final-eval, mean over seeds.
  • Caveat: 2 seeds for many 500k runs — treat single-config differences cautiously; the 1M task4/5 runs used 4 seeds.
